# sbSearchEngine
Search Engine for local text files. Cpp project.

Поисковый движок по локальным текстовым файлам. 
Данная работа является итоговым проектом учебного курса "Разработчик C++".
Поисковый движок осуществляет индексирование текстовых файлов - документов, пути к этим файлам должны быть указаны в config.json. 
Затем осуществляется поиск в индексе фраз, указанных в request.json. Результатом поиска является отчет, в котором каждой поисковой фразе 
соответствует список документов, отсортированный по убыванию относительной релевантности. Отчет сохряняется в файле answer.json.

Файлы config.json, request.json и answer.json лежат в той же директории, что main.cpp, в корневой директории проекта.
Пути к этим файлам указаны в lib/constants.h. Сам же constsants.h формируется из constsnts.h.in сборщиком CMake. См. CMakeLists.txt функция configure_file().
Так что для изменения этих путей нужно редактировать lib/constsnts.h.in.

Движок написан на C++ 17. 
Для работы с JSON файлами применяется библиотека nlohmann/json: JSON for Modern C++ - [GitHub.](https://github.com/nlohmann/json)

В качестве системы тестирования применяется GoogleTest - Google Testing and Mocking Framework -  [GitHub.](https://github.com/google/googletest)

Система сборки CMake версии не ниже 3.24, для работы FetchContent, которым подтягивается GoogleTest из архива на github.com.
Кодировка файлов, подлежащих индексированию, должна быть utf-8. Тексты могут быть написаны как латиницей, так и кириллицей.

В программе реализованы следующие классы.

class ConverterJSON.

Методы класса позволяют получить вектор строк с путями к файлам - документам для индексирования и 
вектор строк поисковых фраз. Для этого методы класса считывают содержимое файлов config.json и reequest.json из корневой директории проекта.

class TextDoc.

Методы класса считывают текстовые файлы в объекты std::string, затем ковертируют их в std::wstring для работы с кириллицей. 
Затем полученная строка разбивается на слова, слова записываются в вектор. 
Также здесь каждому документу присваивается идентификатор. Идентификаторы растут в порядке следования документов в списке в config.json. Первый документ в списке получает ID равный 0.

class InvertedIndex.

Задача этого класса сформировать инвертированный индекс слов всех документов, подвергнутых индексированию.
Конструктор класса вызывает метод UpdateDocumentBase() с аргументом - вектором имен файлов - документов. 
Для каждого файла - документа создается объект класса TextDoc и все эти объекты помещаются в очередь.
Затем очередь разбирается потоками, и подсчет уникальных вхождений каждого слова в документе ведется в своем потоке для каждого дукумента.
Итогом работы метода  UpdateDocumentBase() является словарь, ключом выступает слово, а значением другой словарь, где ключ - это ID документа, 
а значение - количество вхождений этого слова в документ. Это и есть инвертированный индекс.

class Request

Класс для работы с запросами. Здесь строка запроса std::string конвертируется в std::wstring и разбивается на слова.
И ещё каждый запрос получает идентификатор запроса.

class SearchServer

Методами этого класса ведется поиск по индексу и сохранение результата поиска в файл answer.json.
Методы работают с членами класса - структурами: 

RequestRelRaw - структура хранит вектор, в котором каждому слову в запросе соответствует словарь. Ключ - ID запроса.
Значение - словарь из пар ID документа, количество вхождений данного слова.

SortRel - структура хранит финальный результат. Каждому запросу соответствует отсортированная очередь из пар: 
общее количество вхождений слов из запроса в документ с ID. Т.е. абсолютная релевантность. 
Другой член структуры - entryRank - хранит относительную релевантность.

Для заполнения структур служат следующие методы.

MakeSearch(InvertedIndex &) получает аргументом ссылку на объект класса InvertedIndex, то есть доступ к инвертированному индексу.
Для каждого запроса создает свой объект класса Request и помещает эти объекты в вектор.
Затем для каждого запроса и для каждого слова в запросе получает из индекса пары: документ ID - количество вхождений.
Производит суммирование вхождений слов - получает общее количество вхождений всех слов запроса в каждый документ, 
то есть абсолютную релевантность. И заполняет структуру  RequestRelRaw.

MakeSortRel получает аргументом ссылку на структуру RequestRelRaw, производит подсчет относительной релевантности и 
заполняет структуру SortRel.

Метод saveAnswers(SortRel &) производит запись результатов в файл answer.json.

В функции main() сперва вызывается статический метод ConverterJSON::MakeConfig(), затем создается объект класса
InvertedIndex invIdx, и этот объект ссылкой передается статическому методу SearchServer::MakeSearch(invIdx).
В самом методе MakeSearch вызываются методы MakeSortRel и SaveAnswers.
